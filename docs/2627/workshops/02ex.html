Question 5

R will be default treat a variable with only numbers in it as a numeric variable, and anything else as a “character” variable (“character” = any letters/numbers/symbols).

Change the eye colour variable to a factor in R, that only has the options “brown”, “blue”, “green” or “hazel”.

What happens when you now do skim(data)? How does it treat this variable differently?

Hints

Here is the basic code structure for taking a dataframe named mydata and adding a new variable (or changing an existing one).

  • Line 2 takes the data, and passes it to line 3.
  • Lines 3-5 “mutates” the data, adding a variable called new_variable. The new_variable is going to contain the values of an existing variable (old_variable) but with some function applied to them. For example, if we had new_variable = old_variable/2 then it would contain the same values but halved.
  • It feels odd, but it helps to think about the first line of code once we’ve thought of the others. This line re-assigns the name mydata to the mutated mydata, thereby making the change save to the object in R’s memory.
mydata <-
  mydata |> 
  mutate(
    new_variable = function(old_variable)
  )

To make a variable a factor, the function we want is factor(). This will consider every different value that we have to be a possible category:

factor(variable)

If we want to have a specific set of possible values, we can do so with the levels part of the function, replacing the "..." with the options we consider valid:

factor(variable, levels = c("...","...","...", ....... ))

You can see more of this stuff in Flashcard P3 TODO link.

Solution 5. 

sdata <-
  sdata |>
  mutate(
    eyecolour = factor(eyecolour, levels = c("brown","blue","green","hazel"))
  )
